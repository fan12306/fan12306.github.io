  • http.createServer() server.on('request' , funciton() {})但客户端请求发过来，就会自动触发服务器的 request 请求事件，然后执行第二个参数，回调处理
  • server.on('request' , funciton() {})`
    • request 请求事件处理函数，需要接受两个函数；

Node 中的 js

  • Node 为 js 提供了很多服务器级别的 API，这些 API 绝大多数都被包装到了模块中，需要使用 require 进行引用
  • 当运行函数遇到 require 就会跳转到对应的模块中执行相应的代码，然后回来
  • 相对路径要加./
  • 在 Node 中没有全局作用域，只有模块作用域
  • 可以省略后缀名，默认.js
  • require（）方法的两个作用：
    • 拿到文件模块并执行里面的代码；
    • 拿到被加载文件模块到处的接口对象

  • 尽量减少代码的重复

多重边框

灵活的背景定位

  1. background-position 属性允许我们指定背景图片距离任意角的偏移量，只要我们在偏移量前面指定关键字。例如： background-position: right 10px bottom 20px;
  2. background-position 是以 padding-box 为准的，因此 top，left 默认的指的是 padding box 的左上角，我们有一个新属性 border-origin 默认的情况下是 padding-box，我们可以通过改变 content-box 的方法，来改变默认行为；
  3. calc 函数，可以实现 1 中的问题；background-position: calc(100%-20px) calc(100%-10px);

  3. 还可以通过 background-size 来调整其的尺寸；
  4. 垂直条纹需要在开头加上一个额外的参数来制定简便的参数，默认值 to bottom ，通过调整 background 的方向即可

垂直居中

  1. 利用 position absolute，transform： translate（-50% , -50%）
    • 最好不使用 absolute 的，太容易对布局产生影响；

  3. 用 margin： 5vh auto 0；translate： translateY（-50%）

    • 原因 1： 在 v-leave-to 中设置了 translateX（100%）因此页面进去之后出来时要回到原来的那个地方，就会有再回来的动画
    • 解决方法： 分开 v-enter 与 v-leave-to 并将其位置设为 translateX(-100%)即可

    • 原因 3： 目标没有 position: absolute;
    • 解决方法： 设置 position: absolute;

请求后台服务器

  • get()方法

    • 作用：获取服务器数据；
    • 参数
      • 服务器网址；
      • 可选参数；
    • Vue.$http.get()
    • 通过.then 来获取服务器返回的数据返回成功的回调函数。
    • 想要获取成功的回调函数需要 result.body 来获取

  • post()方法

    • 作用：上传数组给服务器；
    • Vue.$http.post()
    • 限制：手动的 post 发送的请求没有表单格式，需要进一步处理，需要配置参数 emulateJSON: true;
    • 参数：
      • 服务器网址；
      • 可选参数 ，需要上传的数据以对象的形式提交；
      • 设置表单的 emulateJSON：true；
    • 通过 then 调取获取成功的函数
    • 想要获取成功的回调函数需要 result.body 来获取

  • jsonp()方法

    • Vue.$http.jsonp()
    • 通过 then 调取获取成功的函数

注意事项

  • 由于需要反复获取地址，维护起来不方便采用根地址形式
    • Vue.http.options.root = /root;
    • 若启用的数据接口的根域名。那么在每次请求的时候需要的相对路径，前面不能带/ ，否则不会启用路径拼接。
  • 每次 post 都要配置 emulateJSON 的话会很繁琐
    • Vue.http.emulateJSON = true;

Vue 动画

  • 进入与退出都有三个阶段
    • v-enter （v-leave）时间点
    • v-enter-active （v-leave-active）时间段
    • v-enter-to （v-leave-to）时间点
    • 需要动画的元素被 transition 包裹起来 transition 是有 Vue 官方提供
    • 为.v-enter or .v-leave-to 设置 CSS 样式就可为
  • 修改 v-前缀
    • 在 transition 内添加 name 属性可以设置独立的 CSS 样式
      • 例如 name: ‘my’ my-enter
  • 钩子函数-动画的生命函数 半场动画
    • 动画的钩子函数表示第一个参数 el 表示要执行动画的 DOM 元素，是原生的 JS DOM 元素 可以认为 el 是通过 document.getElementById()获取到的
      • beforeEnter(el){
        表示动画入场之前，此时动画尚未开始，可设置动画的初始样式
        }
      • enter(el,done){
        表示动画开始之后的样式，可以设置动画完成之后的样式;
        done 就是 afterEnter 函数，调用 done 函数能供解决延时问题
        }
      • afterEnter(el) {
        动画完成之后会调用该函数
        }
  • 在实现列表过渡时，不能使用 transition 包裹，需要使用 transition-group 包裹
    • 必须要为每一个属性设置：key 绑定
    • v-move 与 v-leave-active 配合使用能够使列表元素渐渐离开;
      • v-leave-active = absolute;
    • 给 translation-group 添加一个 appear 能实现缓慢进去；
    • 通过设定 tag 属性制定 translation-group 渲染的元素，默认为 span，

路由

前端路由： 在单页面中通过 hash 改变来切换页面的方式，称之为前端页面;

  • 当导入 Vue.router 包后，在 window 全局对象中就会生出一个路由的构造函数叫做 VueRouter

  • 生成构造函数之后可以为其传递一个配置对象；

  • routes：为配置规则，有两个必须的属性，是一个数组；

    • 属性 1： path ，表示监听哪个路由地址；
    • 属性 2： component ，展示 conponent 对应的组件
      • component 的值必须是一个模板对象，不能是一个组建名称

  • 在实例上有 router，需要在 router 上注册一下构造函数返回的对象
    router: router //router为构造函数返回的对象

  • 是由 Vue 提供的元素，专门用来当作占位符，将来匹配道德路由规则就会展示在 router-view 上

  • 路由的过程

    • 由事件触发网址的改变，进而被 vue 实例中的 router 监听到网站的变化，再而到路由构造函实例寻找相应的匹配规则，成功后，通过相应的组件展示相应的模板

  • radirect 强制执行某个页面

  • 路由选中高亮的两种方法：（由标签 router-link 提供，也就是说要在这里面才能实现高亮）

    • vue 原生的类名.router-link-active 中设置选中时元素高亮
    • 在构造函数内设置一个 linkActiveClass 属性再设置相应的名称

  • 路由规则定义参数

    • 如果在路由中使用查询字符串，给路由传递参数，不需要改变路由规则里的 path 值

    • 通过原生$router.query.（id&name）可以获取相应的参数

    • 通过原生$router.params.id&name 可以获取相应的参数

  • 通过 children 实现路由嵌套

  • 命名化视图

  • watch 属性监听 data 中指定的数据变化，然后触发 watch 中的对应的处理函数

      • newvalue 最新的的数字；
      • oldvalue 次新的数字；
    • 可监听非 DOM 的元素

      watch: '$route.path'(newVal, oldVal) {...}

  • computed 属性

    • 可以定义一些属性，这些属性叫做计算属性，本质就是一个方法，只不过我们在使用这些计算属性的时候把他们的名称直接当作属性来是同的，并不会把计算属性直接当作方法去调用
      • 注意： 极端属性在应用的时候千万不要加（）去调用，直接把她当成属性用就好；
      • 注意： 只要计算属性这个 function 内部所有用到的 data 数据发生变化，就会立即重新计算这个计算属性的值；
      • 注意： 计算属性的的求值结果会被缓存起来；如果计算属性方法中所有的数据都没有发生过变化，不会对计算属性求值；
