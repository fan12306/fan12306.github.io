Posted on 2019-08-14

AJAX

创建XMLHttpRequest对象

  1. 作用： 用于在后台与服务器交流数据，这意味着可以在不重新加载整个网页的情况下，对网页的某部分进行更新
  2. 语法： variable = new XMLHttpRequest()

    向服务器发送请求

  3. 使用XMLHttpRequest对象的open()与send()
  4. 语法： xmlhttp.open('GET' , "ajax_info.text" , true)
    • 规定请求的类型、URL以及是否异步处理请求
      • method： 请求的类型；GET或POST
      • url： 文件在服务器上的位置
      • async： true（异步）或false（同步）
  5. 语法xmlhttp.send()
    • 将请求发送发送到服务器上
      • string： 仅用于POST请求

        GET还是POST

  • 与post相比，GET更简单，并且在大部分情况下都可以使用，在以下情况下使用post
    1. 无法使用缓存文件（更新服务器上的文件或数据库）
    2. 向服务器发送大量数据（POST没有数据数量限制）
    3. 发送包含未知字符的用户输入，POST比GET更加可靠

      url-服务器上的文件

  1. open发放的url参数是服务器文件额地址
    • xmlhttp.open(“GET” , “文件地址”)

      异步为TRUE

  2. 发送的请求是AJAX的时候第三个参数必须为true ，表示异步获取
  3. 如需获得来自服务器的响应，请使用XMLHttpRequest对象的responseText或responseXML属性
    • responseText 获得字符串形式的相应数据
    • responeXML 获得XML形式的相应数据

      onreadystatechange事件

  4. 当请求被发送到服务器是，我们需要执行一些相应的任务，每当ready State改变时，就会触发onreadystatechange事件。
  5. 三个重要属性
    • onreadystatechange： 存储函数或函数名，每当readyState属性改变时，就会触发调用还函数
    • 存有XMLHttpRequest的状态。从0~4发生变化
      • 0：请求未初始化；
      • 1: 服务器已经建立
      • 2： 请求已经接受
      • 3： 请求处理中
      • 4：请求已经完成
    • status
      • 200 : OK
      • 404: 为找到页面

    • 通过then调取获取成功的函数

Vuex

  • Vuex是为了保存组件共享数据而诞生的，如果组件之间又要共享的数据，可以通过直接挂载到vuex中，而不必通过父子组件之间传值了，组建数据不共享不必放入vuex中；
  • Vuex是一个全局共享数据存储区域，相当于一个数据的仓库；
  • 装包：
    1. 运行 npm i vuex -S
    2. 导入包
    3. 注册到Vue中去
    4. new Vuex store() 实例，得到一个数据仓储对象；state是一个对象，相当于实例的的data，mutations也是一个对象，相当于实例的methods
      • 如果在组件中想要访问store中的数据，只能通过this.$store.state.*** 来访问
    5. 将创建的vuex实例挂载到vue实例上，只要挂载到到vue实例上就可以全局访问
  • 如果想要修改state上的数据，只能通过调用this.$state.commit(‘方法名’)来实现；例如: this.$state.commit(“increment”)
    • mutations的函数参数列表中只能传递两个参数，第一个参数为state状态；第二个参数为通过commit提交过来的参数；
    • 若需要提交多个参数，则需要用以对象的形式的方式来进行传参，在进行调用即可；
  • getters是一个对象，里面封装的全部都是函数，只负责对外提供数据，修改请找mutations，类似于过滤器，没有修改数据里面的值，只是提供了一层包装，还与comuputed属性很像，只要state的数据发生改变，getter正好引用了这个数据的话，那么就会立即出发getters的重新求值 ，参数为state，通过对象调用的方式可以获得state中的数据
