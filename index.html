JSONP

JSONP 的概念

  1. 浏览器同源策略的缘故：同一个协议、同一个域名、同一个端口；ajax 中不允许请求非同源的 URL，JSONP 就是来解决这个问题的

JSONP 的原理

  1. ajax 请求受到同源策略的影响，不允许进行跨域请求，而 script 标签 src 属性中的连接却可以访问跨域的 js 脚本，利用这个特性，服务端不再返回 JSON 格式的数据，而是返回一段调用某个函数的 js 代码，在 src 中进行了调用，这样就实现了调用。

后端代理

  1. 因为域名保护，因此我们无法拿到数据，只能通过后端代理来获取数据，前端无法绕过 Host 与 refer

  2. 前端请求修改为自己的 server 端，我们的地址再去请求服务端，通过 axios 请求来修改 Header 中 referer 与 Host 中的数据

  3. 反向代理：本方法是在自己的浏览器创建一个服务器，然后让自己的服务器去请求目标服务器。而且跨域是针对 JavaScript 来说的，JavaScript 是插入 HTML 页面后在浏览器上执行的脚本。服务器之间是可以随便请求数据而不受限制的。我们通过自己创建的服务器去请求目标服务器，然后在从我们客户端去请求我们自己创建的服务器，这就不存在跨域了。

json 和 jsonp 的区别：

  1. json 是一种格式，jsonp 是一种请求跨域资源的方式。

  2. 跨域：是指浏览器不能执行其他网站的脚本，它是由浏览器的同源策略造成的，是浏览器施加的安全限制。在跨域情况下，XMLHTTPRequest 是不能发送异步请求的。
    所谓同源是指域名、协议、端口均相同。

  3. 那么，同是跨域方法，为什么轮播图的请求可以用 jsonp 的方式，而歌单的请求要使用反向代理，两个都是跨域方法。

  1. jsonp 的局限性：只支持 GET 方式的 HTTP 请求，不能解决不同域的两个页面之间如何进行 JavaScript 调用的问题

AJAX

创建XMLHttpRequest对象

  1. 作用： 用于在后台与服务器交流数据，这意味着可以在不重新加载整个网页的情况下，对网页的某部分进行更新
  2. 语法： variable = new XMLHttpRequest()

    向服务器发送请求

  3. 使用XMLHttpRequest对象的open()与send()
  4. 语法： xmlhttp.open('GET' , "ajax_info.text" , true)
    • 规定请求的类型、URL以及是否异步处理请求
      • method： 请求的类型；GET或POST
      • url： 文件在服务器上的位置
      • async： true（异步）或false（同步）
  5. 语法xmlhttp.send()
    • 将请求发送发送到服务器上
      • string： 仅用于POST请求

        GET还是POST

  • 与post相比，GET更简单，并且在大部分情况下都可以使用，在以下情况下使用post
    1. 无法使用缓存文件（更新服务器上的文件或数据库）
    2. 向服务器发送大量数据（POST没有数据数量限制）
    3. 发送包含未知字符的用户输入，POST比GET更加可靠

      url-服务器上的文件

  1. open发放的url参数是服务器文件额地址
    • xmlhttp.open(“GET” , “文件地址”)

      异步为TRUE

  2. 发送的请求是AJAX的时候第三个参数必须为true ，表示异步获取
  3. 如需获得来自服务器的响应，请使用XMLHttpRequest对象的responseText或responseXML属性
    • responseText 获得字符串形式的相应数据
    • responeXML 获得XML形式的相应数据

      onreadystatechange事件

  4. 当请求被发送到服务器是，我们需要执行一些相应的任务，每当ready State改变时，就会触发onreadystatechange事件。
  5. 三个重要属性
    • onreadystatechange： 存储函数或函数名，每当readyState属性改变时，就会触发调用还函数
    • 存有XMLHttpRequest的状态。从0~4发生变化
      • 0：请求未初始化；
      • 1: 服务器已经建立
      • 2： 请求已经接受
      • 3： 请求处理中
      • 4：请求已经完成
    • status
      • 200 : OK
      • 404: 为找到页面

请求后台服务器

  • get()方法

    • 作用：获取服务器数据；
    • 参数
      • 服务器网址；
      • 可选参数；
    • Vue.$http.get()
    • 通过.then来获取服务器返回的数据返回成功的回调函数。
    • 想要获取成功的回调函数需要result.body来获取

  • post()方法

    • 作用：上传数组给服务器；
    • Vue.$http.post()
    • 限制：手动的post发送的请求没有表单格式，需要进一步处理，需要配置参数emulateJSON: true;
    • 参数：
      • 服务器网址；
      • 可选参数 ，需要上传的数据以对象的形式提交；
      • 设置表单的emulateJSON：true；
    • 通过then调取获取成功的函数
    • 想要获取成功的回调函数需要result.body来获取

  • jsonp()方法

    • Vue.$http.jsonp()
    • 通过then调取获取成功的函数

注意事项

  • 由于需要反复获取地址，维护起来不方便采用根地址形式
    • Vue.http.options.root = /root;
    • 若启用的数据接口的根域名。那么在每次请求的时候需要的相对路径，前面不能带/ ，否则不会启用路径拼接。
  • 每次post都要配置emulateJSON 的话会很繁琐
    • Vue.http.emulateJSON = true;

Vue 动画

  • 进入与退出都有三个阶段
    • v-enter （v-leave）时间点
    • v-enter-active （v-leave-active）时间段
    • v-enter-to （v-leave-to）时间点
    • 需要动画的元素被transition包裹起来 transition是有Vue官方提供
    • 为.v-enter or .v-leave-to 设置CSS样式就可为
  • 修改v-前缀
    • 在transition内添加name属性可以设置独立的CSS样式
      • 例如 name: ‘my’ my-enter
  • 钩子函数-动画的生命函数 半场动画
    • 动画的钩子函数表示第一个参数el 表示要执行动画的DOM元素，是原生的JS DOM元素 可以认为el是通过document.getElementById()获取到的
      • beforeEnter(el){
        表示动画入场之前，此时动画尚未开始，可设置动画的初始样式
        }
      • enter(el,done){
        表示动画开始之后的样式，可以设置动画完成之后的样式;
        done就是afterEnter函数，调用done函数能供解决延时问题
        }
      • afterEnter(el) {
        动画完成之后会调用该函数
        }
  • 在实现列表过渡时，不能使用transition包裹，需要使用transition-group包裹
    • 必须要为每一个属性设置：key绑定
    • v-move与v-leave-active 配合使用能够使列表元素渐渐离开;
      • v-leave-active = absolute;
    • 给translation-group添加一个appear能实现缓慢进去；
    • 通过设定tag属性制定translation-group渲染的元素，默认为span，

      • 组件化是从UI界面的角度进行划分，前端的组件化，方便UI组建的重用

    路由

前端路由： 在单页面中通过hash改变来切换页面的方式，称之为前端页面;

  • 当导入Vue.router包后，在window全局对象中就会生出一个路由的构造函数叫做VueRouter

  • 生成构造函数之后可以为其传递一个配置对象；

  • routes：为配置规则，有两个必须的属性，是一个数组；

    • 属性1： path ，表示监听哪个路由地址；
    • 属性2： component ，展示conponent对应的组件
      • component的值必须是一个模板对象，不能是一个组建名称

  • 在实例上有router，需要在router上注册一下构造函数返回的对象
    router: router //router为构造函数返回的对象

  • 是由Vue提供的元素，专门用来当作占位符，将来匹配道德路由规则就会展示在router-view上

  • 路由的过程

    • 由事件触发网址的改变，进而被vue实例中的router监听到网站的变化，再而到路由构造函实例寻找相应的匹配规则，成功后，通过相应的组件展示相应的模板

  • computed属性

    • 可以定义一些属性，这些属性叫做计算属性，本质就是一个方法，只不过我们在使用这些计算属性的时候把他们的名称直接当作属性来是同的，并不会把计算属性直接当作方法去调用
      • 注意： 极端属性在应用的时候千万不要加（）去调用，直接把她当成属性用就好；
      • 注意： 只要计算属性这个function内部所有用到的data数据发生变化，就会立即重新计算这个计算属性的值；
      • 注意： 计算属性的的求值结果会被缓存起来；如果计算属性方法中所有的数据都没有发生过变化，不会对计算属性求值；
