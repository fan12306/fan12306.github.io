      Node中的js

  • 当运行函数遇到require就会跳转到对应的模块中执行相应的代码，然后回来

  • 相对路径要加./

  • 在Node中没有全局作用域，只有模块作用域

  • 可以省略后缀名，默认.js

    • 拿到文件模块并执行里面的代码；
    • 拿到被加载文件模块到处的接口对象

请求后台服务器

  • get()方法

    • 作用：获取服务器数据；
    • 参数
      • 服务器网址；
      • 可选参数；
    • Vue.$http.get()
    • 通过.then来获取服务器返回的数据返回成功的回调函数。
    • 想要获取成功的回调函数需要result.body来获取

  • post()方法

    • 作用：上传数组给服务器；
    • Vue.$http.post()
    • 限制：手动的post发送的请求没有表单格式，需要进一步处理，需要配置参数emulateJSON: true;
    • 参数：
      • 服务器网址；
      • 可选参数 ，需要上传的数据以对象的形式提交；
      • 设置表单的emulateJSON：true；
    • 通过then调取获取成功的函数
    • 想要获取成功的回调函数需要result.body来获取

  • jsonp()方法

    • Vue.$http.jsonp()
    • 通过then调取获取成功的函数

注意事项

  • 由于需要反复获取地址，维护起来不方便采用根地址形式
    • Vue.http.options.root = /root;
    • 若启用的数据接口的根域名。那么在每次请求的时候需要的相对路径，前面不能带/ ，否则不会启用路径拼接。
  • 每次post都要配置emulateJSON 的话会很繁琐
    • Vue.http.emulateJSON = true;

前端路由： 在单页面中通过hash改变来切换页面的方式，称之为前端页面;

  • 当导入Vue.router包后，在window全局对象中就会生出一个路由的构造函数叫做VueRouter

  • routes：为配置规则，有两个必须的属性，是一个数组；

  • computed属性

    • 可以定义一些属性，这些属性叫做计算属性，本质就是一个方法，只不过我们在使用这些计算属性的时候把他们的名称直接当作属性来是同的，并不会把计算属性直接当作方法去调用
      • 注意： 极端属性在应用的时候千万不要加（）去调用，直接把她当成属性用就好；
      • 注意： 只要计算属性这个function内部所有用到的data数据发生变化，就会立即重新计算这个计算属性的值；
      • 注意： 计算属性的的求值结果会被缓存起来；如果计算属性方法中所有的数据都没有发生过变化，不会对计算属性求值；

  • webpack 处理第三方文件类型的过程：

    1. 发现不是js类型的文件就会去到配置文件中查找有没有对应的第三方loader规则；
    2. 如果找到，就会调用对应的loader处理这种文件的类型；
    3. 从后往前调用；
    4. 当最后使用loader调用完毕，会把处理结果交给webpack打包，最终输出到bundle.js中去

  • 关于webpack上传图片的问题

    • 默认情况下webpack无法处理css文件中的url地址，不管是图片还是字体库；
      • limit 给定的值是图片的大小，单位是字节 byte 如果我们引用的图片大于或等于给定的limit值，则不会被转为base64格式的字符串，如果图片小于给定的limit值，则会转
      • name可以设置输出的名字，name:[hash:8]-[name].[[ext]]表示前面带有八位hash值的自定义名字的图片
    • json类型文件中不允许注释

  • webpack默认只能处理一部分的ES6的代码，这是就只能通过第三方的loader，来帮助微博pack处理高两级语法，当第三方loader处理完成后，把高级语法转换低级语法再交给webpack打包到bundle.js

      render: c => c(login);

  • webpack 使用路由，需要额外安装第三方的包

    • 注意：
      • 并不像我们在vue练习一般，直接把其放在所绑定的元素身上，因为组件是通过render渲染出来的，它会清空所控制元素的所有元素，再次渲染自己的元素，所以我们要把包放在render的模板对象里面去才能正常显示
      • 为什么不能放在el中呢？主要是render属性与routers属性不同，<rounter-link to:’./account’>事件触发，被router属性监听到URL的变化，使得其与路由构造函数里的routes匹配规则匹配，发现相同的path，则展示响应的组件到占位符中去
      • 机制与render属性不同

  • 普通的style标签只支持普通的样式，如果想启用scss或者less，需要为style元素，设置lang属性推荐在组件中开启scoped属性

    • 是通过样式选择器实现的

按需导入Mint-UI组件

  • MUI类似于Bootstrap一样提供了一部分的样式与成套的HTML的代码块，MInt-UI真正的组件库，用Vue技术封装出来的成套的组件库只适用于Vue

  • MUI并不能使用npm下载，只要自行到官网下载

    • 压缩的包我们主要需要的是dist与examples，放入新建文件夹名字为lib里去（为手动添加的资源）

  • bootstrap可以在npm中下载

Vuex

  • Vuex是为了保存组件共享数据而诞生的，如果组件之间又要共享的数据，可以通过直接挂载到vuex中，而不必通过父子组件之间传值了，组建数据不共享不必放入vuex中；
  • Vuex是一个全局共享数据存储区域，相当于一个数据的仓库；
  • 装包：
    1. 运行 npm i vuex -S
    2. 导入包
    3. 注册到Vue中去
    4. new Vuex store() 实例，得到一个数据仓储对象；state是一个对象，相当于实例的的data，mutations也是一个对象，相当于实例的methods
      • 如果在组件中想要访问store中的数据，只能通过this.$store.state.*** 来访问
    5. 将创建的vuex实例挂载到vue实例上，只要挂载到到vue实例上就可以全局访问
  • 如果想要修改state上的数据，只能通过调用this.$state.commit(‘方法名’)来实现；例如: this.$state.commit(“increment”)
    • mutations的函数参数列表中只能传递两个参数，第一个参数为state状态；第二个参数为通过commit提交过来的参数；
    • 若需要提交多个参数，则需要用以对象的形式的方式来进行传参，在进行调用即可；
  • getters是一个对象，里面封装的全部都是函数，只负责对外提供数据，修改请找mutations，类似于过滤器，没有修改数据里面的值，只是提供了一层包装，还与comuputed属性很像，只要state的数据发生改变，getter正好引用了这个数据的话，那么就会立即出发getters的重新求值 ，参数为state，通过对象调用的方式可以获得state中的数据
