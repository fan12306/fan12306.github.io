请求后台服务器

  • get()方法

    • 作用：获取服务器数据；
    • 参数
      • 服务器网址；
      • 可选参数；
    • Vue.$http.get()
    • 通过.then来获取服务器返回的数据返回成功的回调函数。
    • 想要获取成功的回调函数需要result.body来获取

  • post()方法

    • 作用：上传数组给服务器；
    • Vue.$http.post()
    • 限制：手动的post发送的请求没有表单格式，需要进一步处理，需要配置参数emulateJSON: true;
    • 参数：
      • 服务器网址；
      • 可选参数 ，需要上传的数据以对象的形式提交；
      • 设置表单的emulateJSON：true；
    • 通过then调取获取成功的函数
    • 想要获取成功的回调函数需要result.body来获取

  • jsonp()方法

    • Vue.$http.jsonp()
    • 通过then调取获取成功的函数

注意事项

  • 由于需要反复获取地址，维护起来不方便采用根地址形式
    • Vue.http.options.root = /root;
    • 若启用的数据接口的根域名。那么在每次请求的时候需要的相对路径，前面不能带/ ，否则不会启用路径拼接。
  • 每次post都要配置emulateJSON 的话会很繁琐
    • Vue.http.emulateJSON = true;

Vue 动画

  • 进入与退出都有三个阶段
    • v-enter （v-leave）时间点
    • v-enter-active （v-leave-active）时间段
    • v-enter-to （v-leave-to）时间点
    • 需要动画的元素被transition包裹起来 transition是有Vue官方提供
    • 为.v-enter or .v-leave-to 设置CSS样式就可为
  • 修改v-前缀
    • 在transition内添加name属性可以设置独立的CSS样式
      • 例如 name: ‘my’ my-enter
  • 钩子函数-动画的生命函数 半场动画
    • 动画的钩子函数表示第一个参数el 表示要执行动画的DOM元素，是原生的JS DOM元素 可以认为el是通过document.getElementById()获取到的
      • beforeEnter(el){
        表示动画入场之前，此时动画尚未开始，可设置动画的初始样式
        }
      • enter(el,done){
        表示动画开始之后的样式，可以设置动画完成之后的样式;
        done就是afterEnter函数，调用done函数能供解决延时问题
        }
      • afterEnter(el) {
        动画完成之后会调用该函数
        }
  • 在实现列表过渡时，不能使用transition包裹，需要使用transition-group包裹
    • 必须要为每一个属性设置：key绑定
    • v-move与v-leave-active 配合使用能够使列表元素渐渐离开;
      • v-leave-active = absolute;
    • 给translation-group添加一个appear能实现缓慢进去；
    • 通过设定tag属性制定translation-group渲染的元素，默认为span，

Vue组件化

  • 组件的含义：

    • 组建的出现就是为了拆分Vue实例的代码量，能够让我们以不同的组件，来划分不同功能模块
    • 组件化与模块化的区别
      • 模块化是为了后台开发是分层开发，保证每个功能职能专一；
      • 组件化是从UI界面的角度进行划分，前端的组件化，方便UI组建的重用

    路由

前端路由： 在单页面中通过hash改变来切换页面的方式，称之为前端页面;

  • 当导入Vue.router包后，在window全局对象中就会生出一个路由的构造函数叫做VueRouter

  • 生成构造函数之后可以为其传递一个配置对象；

  • routes：为配置规则，有两个必须的属性，是一个数组；

    • 属性1： path ，表示监听哪个路由地址；
    • 属性2： component ，展示conponent对应的组件
      • component的值必须是一个模板对象，不能是一个组建名称

  • 在实例上有router，需要在router上注册一下构造函数返回的对象
    router: router //router为构造函数返回的对象

  • 是由Vue提供的元素，专门用来当作占位符，将来匹配道德路由规则就会展示在router-view上

  • 路由的过程

    • 由事件触发网址的改变，进而被vue实例中的router监听到网站的变化，再而到路由构造函实例寻找相应的匹配规则，成功后，通过相应的组件展示相应的模板
  •  <router-link to="/login?id=10&name=fan">登陆</router-link>

    • 通过给添加name属性即可
    • watch属性监听data中指定的数据变化，然后触发watch中的对应的处理函数

        watch: '$route.path'(newVal, oldVal) {...}

  • computed属性

    • 可以定义一些属性，这些属性叫做计算属性，本质就是一个方法，只不过我们在使用这些计算属性的时候把他们的名称直接当作属性来是同的，并不会把计算属性直接当作方法去调用
      • 注意： 极端属性在应用的时候千万不要加（）去调用，直接把她当成属性用就好；
      • 注意： 只要计算属性这个function内部所有用到的data数据发生变化，就会立即重新计算这个计算属性的值；
      • 注意： 计算属性的的求值结果会被缓存起来；如果计算属性方法中所有的数据都没有发生过变化，不会对计算属性求值；
