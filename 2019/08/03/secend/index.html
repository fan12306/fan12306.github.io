请求后台服务器

  • get()方法

    • 作用：获取服务器数据；
    • 参数
      • 服务器网址；
      • 可选参数；
    • Vue.$http.get()
    • 通过.then来获取服务器返回的数据返回成功的回调函数。
    • 想要获取成功的回调函数需要result.body来获取

  • post()方法

    • 作用：上传数组给服务器；
    • Vue.$http.post()
    • 限制：手动的post发送的请求没有表单格式，需要进一步处理，需要配置参数emulateJSON: true;
    • 参数：
      • 服务器网址；
      • 可选参数 ，需要上传的数据以对象的形式提交；
      • 设置表单的emulateJSON：true；
    • 通过then调取获取成功的函数
    • 想要获取成功的回调函数需要result.body来获取

  • jsonp()方法

    • Vue.$http.jsonp()
    • 通过then调取获取成功的函数

注意事项

  • 由于需要反复获取地址，维护起来不方便采用根地址形式
    • Vue.http.options.root = /root;
    • 若启用的数据接口的根域名。那么在每次请求的时候需要的相对路径，前面不能带/ ，否则不会启用路径拼接。
  • 每次post都要配置emulateJSON 的话会很繁琐
    • Vue.http.emulateJSON = true;

Vue 动画

  • 进入与退出都有三个阶段
    • v-enter （v-leave）时间点
    • v-enter-active （v-leave-active）时间段
    • v-enter-to （v-leave-to）时间点
    • 需要动画的元素被transition包裹起来 transition是有Vue官方提供
    • 为.v-enter or .v-leave-to 设置CSS样式就可为
  • 修改v-前缀
    • 在transition内添加name属性可以设置独立的CSS样式
      • 例如 name: ‘my’ my-enter
  • 钩子函数-动画的生命函数 半场动画
    • 动画的钩子函数表示第一个参数el 表示要执行动画的DOM元素，是原生的JS DOM元素 可以认为el是通过document.getElementById()获取到的
      • beforeEnter(el){
        表示动画入场之前，此时动画尚未开始，可设置动画的初始样式
        }
      • enter(el,done){
        表示动画开始之后的样式，可以设置动画完成之后的样式;
        done就是afterEnter函数，调用done函数能供解决延时问题
        }
      • afterEnter(el) {
        动画完成之后会调用该函数
        }
  • 在实现列表过渡时，不能使用transition包裹，需要使用transition-group包裹
    • 必须要为每一个属性设置：key绑定
    • v-move与v-leave-active 配合使用能够使列表元素渐渐离开;
      • v-leave-active = absolute;
    • 给translation-group添加一个appear能实现缓慢进去；
    • 通过设定tag属性制定translation-group渲染的元素，默认为span，

Vue组件化

  • 组件的含义：

    • 组建的出现就是为了拆分Vue实例的代码量，能够让我们以不同的组件，来划分不同功能模块
    • 组件化与模块化的区别
      • 模块化是为了后台开发是分层开发，保证每个功能职能专一；
      • 组件化是从UI界面的角度进行划分，前端的组件化，方便UI组建的重用

      render属性

  • 是一个函数，形参为提供好的方法叫做 createElement，调用它可以把制定的模块喧嚷为html结构

  • return的结构是代替被el绑定的那个容器

  • 与组件的区别：

    • 组件并不会代替那个容器，只是相当于一个插件表达式；
    • render会清空所有容器里面的元素，再代替它

    路由

前端路由： 在单页面中通过hash改变来切换页面的方式，称之为前端页面;

  • 当导入Vue.router包后，在window全局对象中就会生出一个路由的构造函数叫做VueRouter

  • 生成构造函数之后可以为其传递一个配置对象；

  • routes：为配置规则，有两个必须的属性，是一个数组；

    • 属性1： path ，表示监听哪个路由地址；
    • 属性2： component ，展示conponent对应的组件
      • component的值必须是一个模板对象，不能是一个组建名称

  • 在实例上有router，需要在router上注册一下构造函数返回的对象
    router: router //router为构造函数返回的对象

  • 是由Vue提供的元素，专门用来当作占位符，将来匹配道德路由规则就会展示在router-view上

  • 路由的过程

    • 由事件触发网址的改变，进而被vue实例中的router监听到网站的变化，再而到路由构造函实例寻找相应的匹配规则，成功后，通过相应的组件展示相应的模板

  • 在package.json中script’dev’添加“webpack-dev-server”

  • 如果工具想要正常运行，必须还要在本地中运行在本地项目中，必须安装webpack

  • webpack-dev-server帮我们打包生成的bundle。js文件，并没有存放到实际的为u里磁盘而是直接托管到了电脑的内存中，所以我们在项目根目录根本找不到这个打包好的bundle.js

  • 在dev中传参–open自动打开页面 –port3000设置端口为3000 –content src 以src作为根路径现实与页面

  • hot 热重载 减少不必要代码的更新，实现浏览器的无刷新

  • 当使用html-webpack-plugin

    • 自动在内存中根据指定页面生成一内存的页面；
    • 自动打包好的bundle.js追加到页面中去

  • webpack默认只能打包处理js类型的文件，无法处理其他非js类型的文件；

    • 处理非js类型的文件，需要安装cnpm i style-loader css-loader -D
    • 打开webpack.config.js 这个配置文件，在里面新增一个配置节点，叫做module，他是一个对象；在这个module对象身上有一个rules属性，是一个数组，存放了所有第三方文件的匹配和匹配规则

  • webpack 处理第三方文件类型的过程：

    1. 发现不是js类型的文件就会去到配置文件中查找有没有对应的第三方loader规则；
    2. 如果找到，就会调用对应的loader处理这种文件的类型；
    3. 从后往前调用；
    4. 当最后使用loader调用完毕，会把处理结果交给webpack打包，最终输出到bundle.js中去

  • 关于webpack上传图片的问题

    • 默认情况下webpack无法处理css文件中的url地址，不管是图片还是字体库；
      • limit 给定的值是图片的大小，单位是字节 byte 如果我们引用的图片大于或等于给定的limit值，则不会被转为base64格式的字符串，如果图片小于给定的limit值，则会转
      • name可以设置输出的名字，name:[hash:8]-[name].[[ext]]表示前面带有八位hash值的自定义名字的图片
    • json类型文件中不允许注释

  • webpack默认只能处理一部分的ES6的代码，这是就只能通过第三方的loader，来帮助微博pack处理高两级语法，当第三方loader处理完成后，把高级语法转换低级语法再交给webpack打包到bundle.js

  • 通过BAbel。可以帮助我们将高级的语法转换为低级语法

  • 两套安装Bable功能

    • 第一套 npm i Babel-core babel-loader babel-plugin-transform-runtime -D
    • 第二套 npm i bable-preset-env bable-preset-stage-0 -D
    • 注意
      • 在配置bable的loader规则的时候，必须把node_modules目录，通过exclude选项排除：原因：
        • 如果不排除node_modules,则babel会把node_modules中所有的第三方wjs文件都都打包编译，会非常消耗cpu
        • 即使打包完成无法正常运行
          在项目根目录次啊新建的.babelrc配置文件属于JSON格式，不能书写注释，字符串要双引号
