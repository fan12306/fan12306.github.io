项目开发

  • 项目分析

    • 分析页面组成

      • 页面分为头部、主体、底部三个部分；
      • 头部是固定不动的Header –在Mint-UI上有现成的代码；
      • 底部是一般的界面但是在Mint——UI上的不符合需求，在MUI上有合适的样式

    • 步骤：

      1. 要一直有的概念就是Vue框架基本不用操作DOM，一切在页面渲染出来的界面都是从render函数的template模板中的；

      2. 因为模块是独立的，凡是应用到的框架或样式都需要在main.js中引进，无论Vue还是css；

      3. MUI无法通过npm下载只能从下载下来，但是Mint-UI可以

      4. 所需要的图标样式可以寻找要确保其所用到的样式都能下载下来;

      5. 改造 tabBar 为router-link

      6. 组件需要安装好，并且能都导入到main.js内部，还需要在指定的router模板对象中注册，指定的模板对象也哟啊导入vue-router组件，还需要暴露出去（export default router）

      7. 定义自己的组件一定记得import出去，因为模块是独立的没有根据的组件或则其他都会报错

      8. 轮播图从Mint-UI上查找，匹配进去之后他的高度还需要自行设计；

      9. npm 安装 Vue-resource 向网页获取数据（照片），还需要import与Vue.use()注册，才能通过命令this.$http.get(“网址”)
        获取成功的标志是

      10. 将保存的图片放入一个新数组中，在v-for渲染到页面中去

        • 注意： 需要绑定img或着url其中一个，因为他们都是唯一的；
        • 注意： 需要绑定src，表示这是一个 js 表达式，并不是普通字符串，要在src前加: 绑定后才能正常使用

      11. 样式的选择很重要，在源代码上查找，并找到合适的class类名，设置合适的样式；

      12. 加入动画：

        • 包裹住router-view占位元素
        • 注意：
          • 问题1： 图片先进去再回去；
          • 原因1： 在v-leave-to中设置了translateX（100%）因此页面进去之后出来时要回到原来的那个地方，就会有再回来的动画
          • 解决方法： 分开v-enter与v-leave-to并将其位置设为translateX(-100%)即可
          • 问题2： 图片下方出现滚动条问题；
          • 原因2： 再点击更换页面时，router检测到网址的改变，会瞬间展示出来对应的组件，因此页面的实际宽度就大于当前页面的宽度产生滚动条；
          • 解决方法： 设置属性overflowX: hidden;* 一定要设置x轴方向上的hidden *
          • 问题3： 图片先在下面再上去；
          • 原因3： 目标没有position: absolute;
          • 解决方法： 设置position: absolute;

        1. 时间过滤器：

          • 先安装插件：npm i moment

        2. 获取新闻详情

          • 需要要为每个新闻绑定一个id，因为我们要去的地址带有js表达式，因此需要绑定<route-link :to="'/home/newInfo/' + item.id"></route-link>
            绑定to
          • 获取id $router.params.id
          • 需要在routes配置规则上传入参数id path: /home/newInfo/:id

        3. 提交评论

          • 先校验评论内容是否为空this.mes.trim().length === 0
          • 通过post命令发送评论给服务器
            • 参数1: 地址;
            • 参数2: 提交给服务器的数据对象{ content: this.mes }
            • 参数3: 提交内容的格式;可全局配置{ emulateJSON:true }
          • 拼接数据对此昂实现客户端手动更新
            • 若调用getComment会pageIndex == 2的,那么第一页的评论内容就消失了换一种思路:通过请求成功后,手动拼接对象通过unshift方法追加到评论的第一位置,从而实现到刷新的功能;

        4. 轮滑条无法正常使用

          • webpack关闭严格模式

        5. 图片进行懒加载的效果

          • 引入组件 在mint-UI上引入,并且使用
          • 通过点击事件获得相应的数组列表,需要为方法传递一个参数:表示需要那个列表
          • 一开始就要渲染出来第一页,因此传参数为0;
          • Mint-ui为我们提供了图片的样式,样式一定要按照规则一步一步往下写;
          • 若果运用按需导入,懒加载没有加载的圈圈,因此选用全部加载.

        6. 当使用router-link的时候界面的样式乱了,可能是对应的标签名更换了,因为router-link默认的标签为a标签

        7. 在网页中有两种跳转方式,一种是a标签,还有一种是 window.location.href

        8. 区分this.$route是参数对象,所有的路由参数,params,query都属于它;this.$router是一个路由导航对象,用它可以实现路由的前进后退,跳转到新的URL对象

        9. 抽离子组件,注意之前父组件的属性,还要使用子组件的父组件的的属性是否一致,如果不一致,需要通过循环的方式为其添加相应的属性

          • 若之前的父组件与先有组父件的样式冲突,定义一个属性,让使用轮播图的调用者手动决定是否需要使用之前的样式;
          • 需要在父组件决定是否需要该组件,true or false ,再由子组件接受props绑定class对象决定某样式的样式

        10. 实现img:width100%,需要去除scope;

    • 注意

      • 一定一定不要写错代码，特别是后缀名，会有难以处理的问题

      • 导入样式的方法与导入vue的不一样；

      • Mint-UI是按需导入,导入相应的组件还需要注册

      • 关于高亮的问题，要是Mint—UI内部自带高亮，则可以利用组件的linkActiveClass属性来覆盖原有的样式，从而达到正常的需求；

      • 关于scss的子代选择器

      • 获取通过id获取新闻资讯，记得获取文件的类型。现在是一个数组但是新闻资讯是一个对象所以使用this.NewsInfo = result.body.message[0] NewsInfo是一个对象，再通过对象来调取相应的数据；

      • 子组件导入

        • 子组件在父组件的components属性下注册,记得要先导入;
        • 父组件向子组件传值
          • 在父组件内,子组件的占位符上绑定一个参数,在子组件上使用props注册一下,该属性是一个数组

      • 注意变量的使用;

      • 为了防止数据覆盖,让老数据与新数据使用一个concat方法拼接在一起;

        • this.setCmt = this.setCmt.concat(result.body.message)

      • 刚进入图片分享界面的时候，滚动条无法正常使用，分析后，如果要进入初始化滚动条必须等DOM元素加载完毕，所以将滚动条放在了mounted生命周期钩子函数里；

      • 当滑动条的ok后，发现tabBar无法正常工作，原因可能是，mint里面的样式与tabTar的样式冲突了，现在是把tabBar的样式复制一份换一个名字；（但是我找不全名字）

      • 当样式不符合要求的时候，需要进行绑定处理class时，通过id进行关键绑定<a :class="['mui-control-item' , item.id == 0 ? 'mui-active' : '']"；

      • 解决经典的两边布局

      • 关于props的父组件箱子组将传值与子路由的绑定之间的区别

        • 父组件箱子组建传值需要每个父组件绑定一个属性值;子组件通过props接收到值;
        • 子路由与父路由,子路有需要在父路由上注册,绑定相应的id即可;

      • 小球动画属于半场动画因此使用钩子函数
