项目开发

项目分析

  • 分析页面组成

    • 页面分为头部、主体、底部三个部分；
    • 头部是固定不动的 Header –在 Mint-UI 上有现成的代码；
    • 底部是一般的界面但是在 Mint——UI 上的不符合需求，在 MUI 上有合适的样式

  • 步骤：

    1. 要一直有的概念就是 Vue 框架基本不用操作 DOM，一切在页面渲染出来的界面都是从 render 函数的 template 模板中的；

    2. 因为模块是独立的，凡是应用到的框架或样式都需要在 main.js 中引进，无论 Vue 还是 css；

    3. MUI 无法通过 npm 下载只能从下载下来，但是 Mint-UI 可以

    4. 所需要的图标样式可以寻找要确保其所用到的样式都能下载下来;

    5. 改造 tabBar 为 router-link

    6. 组件需要安装好，并且能都导入到 main.js 内部，还需要在指定的 router 模板对象中注册，指定的模板对象也要导入 vue-router 组件，还需要暴露出去（export default router）

    7. 定义自己的组件一定记得 import 出去，因为模块是独立的没有根据的组件或则其他都会报错

    8. 轮播图从 Mint-UI 上查找，匹配进去之后他的高度还需要自行设计；

    9. npm 安装 Vue-resource 向网页获取数据（照片），还需要 import 与 Vue.use()注册，才能通过命令 this.$http.get(“网址”)
      获取成功的标志是

    10. 将保存的图片放入一个新数组中，在 v-for 渲染到页面中去

      • 注意： 需要绑定 img 或着 url 其中一个，因为他们都是唯一的；
      • 注意： 需要绑定 src，表示这是一个 js 表达式，并不是普通字符串，要在 src 前加: 绑定后才能正常使用

    11. 样式的选择很重要，在源代码上查找，并找到合适的 class 类名，设置合适的样式；

    12. 获取新闻详情

      • 需要要为每个新闻绑定一个 id，因为我们要去的地址带有 js 表达式，因此需要绑定<route-link :to="'/home/newInfo/' + item.id"></route-link>
        绑定 to
      • 获取 id $router.params.id
      • 需要在 routes 配置规则上传入参数 id path: /home/newInfo/:id

    13. 轮滑条无法正常使用

      • webpack 关闭严格模式

    14. 图片进行懒加载的效果

      • 引入组件 在 mint-UI 上引入,并且使用
      • 通过点击事件获得相应的数组列表,需要为方法传递一个参数:表示需要那个列表
      • 一开始就要渲染出来第一页,因此传参数为 0;
      • Mint-ui 为我们提供了图片的样式,样式一定要按照规则一步一步往下写;
        • 若果运用按需导入,懒加载没有加载的圈圈,因此选用全部加载.

    15. 当使用 router-link 的时候界面的样式乱了,可能是对应的标签名更换了,因为 router-link 默认的标签为 a 标签

    16. 实现 img:width100%,需要去除 scope;

区分

  • this.$route 是参数对象,所有的路由参数,params,query 都属于它,拥有着 id 以及很多的属性;
  • this.$router 是一个路由导航对象,用它可以实现路由的前进后退,跳转到新的 URL 对象

  • 若之前的父组件与先有组父件的样式冲突,定义一个属性,让使用轮播图的调用者手动决定是否需要使用之前的样式;
  • 需要在父组件决定是否需要该组件,true or false ,再由子组件接受 props 绑定 class 对象决定某样式的样式

解决框架之间的样式冲突

  • 分析
    1. 刚进入图片分享界面的时候，滚动条无法正常使用；
    2. 当滑动条的 ok 后，发现 tabBar 无法正常工作，原因可能是，mint 里面的样式与 tabTar 的样式冲突了；
    3. 当同一个组件用的同一个样式值不同时；
  • 解决方案：
    1. 如果要进入初始化滚动条必须等 DOM 元素加载完毕，所以将滚动条放在了 mounted 生命周期钩子函数里；
    2. 现在是把 tabBar 的样式复制一份换一个名字；（但是我找不全名字）
    3. 当样式不符合要求的时候，需要进行绑定处理 class 时，通过 id 进行关键绑定<a :class="['mui-control-item' , item.id == 0 ? 'mui-active' : '']"；

解决经典的两边布局

关于 props 的父组件向子组将传值与子路由的绑定之间的区别

- 父组件箱子组建传值需要每个父组件绑定一个属性值;子组件通过props接收到值;
- 子路由与父路由,子路有需要在父路由上注册,绑定相应的id即可;

加入动画

  • 包裹住 router-view 占位元素
  • 注意：
    • 问题 1： 图片先进去再回去；
    • 原因 1： 在 v-leave-to 中设置了 translateX（100%）因此页面进去之后出来时要回到原来的那个地方，就会有再回来的动画
    • 解决方法： 分开 v-enter 与 v-leave-to 并将其位置设为 translateX(-100%)即可
    • 问题 2： 图片下方出现滚动条问题；
    • 原因 2： 再点击更换页面时，router 检测到网址的改变，会瞬间展示出来对应的组件，因此页面的实际宽度就大于当前页面的宽度产生滚动条；
    • 解决方法： 设置属性 overflowX: hidden;_ 一定要设置 x 轴方向上的 hidden _
    • 问题 3： 图片先在下面再上去；
    • 原因 3： 目标没有 position: absolute;
    • 解决方法： 设置 position: absolute;

小球动画的优化

  • 分析： 1. 因为对于不同的的分辨率或者长度不同的数据,小球的 translate 写死了就不能满足需求；
  • 解决方法: 1. 利用 domObject.getBoundingClientRect()可以获取元素的横纵坐标; 2. 通过两个位置的差值来获取准确的数据 3. 由于购物车不属于组件内的数据,因此打算稍稍利用 dom 来获取它位置
  • 注意：
    1. 小球动画属于半场动画因此使用钩子函数

实现加入购物车时候获得选择的数量

    1. 按键属于父组件的，数字属于子组件的；

    3. 涉及子组件向父组件传值，事件调用机制；
  • 解决方案：
    1. 本质： 父组件创建一个方法，子组件调用，并且传值回到父组件即可；
    2. 通过 ref 在标签上应用，可以使用 this.$refs.定义的组件.要拿到的 DOM 的属性，作为值传给父组件；

获取最大库存量

    2. 问题是我们不知道什么时候能狗拿到 max 值，但是我们总能有一时刻能够获取；
  • 解决方案：
    1. 因此我们使用 watch 属性，监听 max 值得变化

如何从购物中获取商品的数量

    4. 之后使用$store.getters.getGoodsCount[item.id]来获取 count，通过父子传值即可实现。

按键的实现

  • 分析

    1. 通过 true 或者 false 来决定开关，通过 id 来确认是什么商品；
    2. 再次进入界面时，按键默认为 true，需要根据以前额历史来确定

  • 解决方案：

    1. 那么可以将其拼接为一个以 id 为属性名，以 selected 的值为属性值，拼接出一个对象；
    1. 调用 this.$store.getters.getAllSelected[item.id]就可以获得 select 的值，绑定到官方提供的 v-model 中，但是并没有起到双向数据绑定；
    2. 通过 change 事件监听到其 value 的改变，并且通过
      this.$store.commit(id , this.$store.getters.getAllSelected[item.id])
      来实现更新 store 与 localStorage 的数据更新
      val为形参，代表this.$store.getters.getAllSelected[item.id]

实现通过按键来决定总价；

  • 分析
    1. 只有当按键的值为 true 的时候，才被选中加入总价；
    2. 按键有一个双向数据绑定，但是病不起作用，需要时刻监听按键的变化；
  • 解决方案：
    1. 利用 jq 里面的 change 事件，时刻监听按键值的变化，再在 store 中定义一个 getters 函数，监听总价的变化，并且时刻更新 localStorage 的数据与 store 的数据；
    2. 一旦按键的值改变，就会触发 change 事件的响应，并立即调用在 getters 定义的函数，计算出总价的变化，并且更新 store 与 localStorage 的数据；

返回键的实现：

  • 分析：
    1. 通过什么方式能够使按键每次触发时能根据历史纪录返回上一页；
    2. 当再次刷新界面时，如何还能实现功能；
    3. 在主页不展示返回键;
    4. 如何监听 url 的地址的变化；
  • 解决方案：
    1. 通过 mint-ui 添加返回键的样式；
    2. 通过点击事件为其绑定一个方法，利用 this.$route.path 的方法来判断路由的地址；
    3. 通过 watch 监听 url 地址的变化；

遇到的问题

  1. 用到路由时,需要使用引入 vue-router 并且在 vue 上面注册还要在实例上面注册;

  2. 动画的问题: 需要把 v-写在父元素外面才能生效;

  3. 时间过滤器：

    • 需要装包；npm i moment -S
    • 需要在全局中导入 moment；
    • 如果直接调用 moment 的话，就是当前的时间；想格式化当前给定的时间，需要给 moment 传递参数；
    • format 需要参数表示以什么方式展示；

  4. 传递 id；

    • 因为在 newsInfo 上需要 id 表明哪个是哪个新闻，所以我们就要从 newList 上获取路径的 id
    • 通过绑定 to 的方式向另外的路由传递 id；这个与父子路由值完全不同；

  5. 啥时候要用 message[0];

    • message 中有很多信息，并且我们只需要其中的某一条；
    • 或者是通过 id 传值；

  6. 制作评论子组件：很关键

    • 获取评论信息： id 是从父组件传递下来的，用 props 接受；
    • 发表评论： 需要判断评论内容是否为空，post 的方法发送评论：
      1. 需要三个参数 网址 ， 对象为 content ， 还有表格格式
      2. 如果回调成功，那个我就制造一个对象传入数据的第一个；

  7. 传递 id 的几种方法；

    1. 需要文中的 id，则可以通过事件传递参数的方式获取对方的 id；
    2. 父传子的获取 id；需要通过数据绑定，在子组件需要用 props 接受；
    3. 子传父获取 id，需要进行事件绑定，在子组件调用事件，并且通过$emit 来传毒参数；
    4. 需要获取上一个组建的 id 需要通过绑定：to 的方式，在路由上面传递参数；并且要在自己的路由上定义一下 id 以获取它；

  8. 应用样式需要 js 表达式的时候

    1. 为 class 类进行绑定；并且加上中括号，可以直接用三元表达式判断；
    2. 需要判断该类是否要应用的时候需要用对象的方式来绑定；

  9. 获取元素的位置信息

    1. 通过 ref 绑定一个参数，之后调用 this.$refs.(绑定的那个值).getBoundingClientRect()即可获取位置信息；
    2. 若是要获取的元素不在本页面，可以使用 DOM 的方法获取，还是调用 getBoundingClientRect()方法；

  10. 子传父，

    1. 在父组件通过调用一个函数，在子组件也调用一个函数
    2. 但是子组件的函数以什么形式调用很关键，什么时候调用，调用会影响什么，一切都要想清楚；
    3. 目前这个项目子组件的方法调用是监听数字盒子的改变，利用 change 函数监听，并且绑定了一个事件，这个事件就是子组件需要返回的函数

  11. vuex

    1. 传值也是需要知道需要传递那些数据，需要通过 mutations 来修改 store 的数据，调用 mutations 需要使用 this.$store.commit(‘***‘ , {})

    2. getters 相当于 filter 与 watch，可以实现数据实时更新，可以通过 this.$store.getters.***()来调用

    3. 一定要学会拼接对象的方法，这个方法很关键；

    4. 删除要涉及的是三个方面的删除；页面上的删除，Vuex 上 store 的数据删除；本地存储的删除

      1. 删除是根据索引值删除的

    5. 按键功能实现：

      1. 绑定在 v-model 上，利用$store.getters.getSelected[item.id]来获取 selected，但是 v-model 不能起到双数据绑定的作用，也就是说不能狗通过点击按键来实现更新 store 的数据与改变 localStorage 的数据，因此还需要通过 mutations 来实现，
      • 问题来了，什么时候调用函数来实现传递数据给 mutations 呢？利用 change 事件监听按键的状态改变，一旦改变的话，就通过时间绑定将数据传递给 mutations 进而实现 selected 的实时更新
